Git Használati Lépések

Git Kód Mit csinál Használati Példa Alternatív Parancsok Gyakori Hibák / Figyelmeztetések
git init Inicializálja a Git repository-t a projekt mappában. Használható egy új projekt indításához: git init Ha nem akarod hozzáadni az összes fájlt, próbálhatod: git add -A Ha a git init parancsot már lefuttattad, a második futtatás felülírhatja a repository-t.
git clone <repository-url> Lemásolja egy távoli repository-t a helyi gépre. Lemásolás egy távoli repo-ból: git clone https://github.com/username/project.git Ha csak egy fájlt szeretnél klónozni, adj meg egy specifikus fájlt is: git clone https://github.com/username/project.git --depth 1 Ha hibát kapsz a clone parancs használata közben, ellenőrizd, hogy a repository URL-je helyes-e.
git status Megmutatja a projekt állapotát (milyen változtatások történtek). Megtekintheted a változásokat: git status Másik módja a status megtekintésének: git diff A status parancs futtatása után biztosítsd, hogy a fájlok helyes állapotban legyenek a commit előtt.
git add . Hozzáadja a változtatásokat a staging area-hoz, hogy készen álljanak a commit-ra. Hozzáadott fájlok az összeshez: git add . Másik módszer a fájlok hozzáadására: git add -p Ha a git add parancsot használod, és nem figyelsz, véletlenül olyan fájlokat is hozzáadhatsz, amik nem szükségesek.
git commit -m 'commit üzenet' Elmenti a változtatásokat a helyi repository-ba egy üzenet kíséretében. Commit a változásokkal: git commit -m 'Első commit' A commit üzenetet módosíthatod a git commit --amend paranccsal. A commit üzenet megadása nélküli commit hibát okozhat: git commit -m ''
git push Feltölti a helyi változtatásokat a távoli repository-ba. Feltöltés a távoli repóba: git push origin master A push parancsot specifikus branch-ra is használhatod: git push origin <branch-name> Ha a push nem sikerül, előfordulhat, hogy a távoli repository-val eltérés van (például mások is dolgoztak rajta).
git pull Letölti a távoli repository változtatásait a helyi gépre. Pull a távoli változások letöltése: git pull Pull helyett frissítheted a távoli ágakat: git fetch A pull parancs gyakran merge konfliktusokat okozhat, ha a helyi és távoli változások nem összeegyeztethetőek.
git branch Megmutatja a repository-ban elérhető ágakat. Ágak megtekintése: git branch Ha új ágat szeretnél létrehozni: git branch <branch-name> Ha nem találod a megfelelő ágat a git branch parancs alatt, akkor új ágat kell létrehoznod.
git checkout <branch-name> Átvált egy másik ágra a munkafolyamat folytatásához. Átváltás egy másik ágra: git checkout develop Ha egy ágra szeretnél váltani anélkül, hogy elköteleznéd a változtatásokat: git checkout -f Ha a checkout után nem változik az ág, akkor próbáld meg a változtatásokat elkötelezni a munka folytatásához.
git merge <branch-name> Összevon két ágat egyesítve a változtatásokat. Merge a feature ágra: git merge feature-xyz A merge helyett próbálhatod a rebase-t: git rebase <branch-name> A merge során előfordulhatnak konfliktusok, amelyeket manuálisan kell megoldani.
git log Megmutatja a commitok történetét, hogy nyomon követhessük a projekt előzményeit. Commitok előzményeinek megtekintése: git log A log helyett próbálhatod a git reflog parancsot, hogy visszakövethesd a mozgásokat. A git log parancs csak a commitokat mutatja, nem minden történeti adatot.
git fetch Frissíti a távoli repository-t anélkül, hogy automatikusan beolvasztaná a változásokat. Frissítést végezhetsz távoli repository-ból: git fetch Alternatívaként használhatod a git pull --rebase-t, hogy a helyi változtatásokat a távoli változtatások elé helyezd. Ha a fetch után nem látod az új változtatásokat, ellenőrizd, hogy a megfelelő távoli repository-t frissítetted-e.
git reset Visszaállítja a staging area-t vagy a commit állapotát egy korábbi állapotra. Visszaállítás egy előző commit állapotára: git reset --hard HEAD~1 Ha nem szeretnéd teljesen visszaállítani a változtatásokat, használhatod a git reset --soft parancsot. A reset parancs elveszítheti a változtatásokat, ha nem használsz `--soft` vagy `--mixed` módot, ezért fontos biztonsági másolatot készíteni.
git diff Megmutatja a fájlok közötti különbségeket. Különbségek megtekintése egy fájl között: git diff Használhatod a git diff --staged parancsot a staging area-ban lévő változtatások megtekintésére. Ha túl sok változást látsz, próbálj szűrni a fájlokra vagy specifikus commitokra.
git rm Eltávolít egy fájlt a repository-ból. Fájl törlésének megoldása: git rm fájlneve.txt Ha nem akarod eltávolítani a fájlt a helyi rendszerből, használhatod a git rm --cached parancsot. Ne használd a git rm parancsot, ha csak a fájlok törlését szeretnéd a staging area-ból.
git tag Lehetővé teszi a verziók megjelölését. Tag létrehozása egy commit-nál: git tag v1.0 Tagek listázása: git tag -l Ha hibás tagot hoztál létre, törölheted azt a git tag -d v1.0 paranccsal.
git stash Ideiglenesen elmenti a nem commitolt változtatásokat, hogy később vissza tudd őket hozni. Változtatások elmentése: git stash Ha egyesíteni akarod a változtatásokat, használd a git stash pop parancsot. Ne hagyd sokáig stash-ben a változtatásokat, mert könnyen elfelejtheted őket.
git clean Törli a nem nyomon követett fájlokat a projektből. Fájlok törlése: git clean -f Használhatod a git clean -fd parancsot a mappák törlésére is. Ne használd, ha nem biztos, hogy törölni szeretnéd a fájlokat.
git bisect Segít megtalálni, hogy mely commit okozott egy hibát. Hiba keresése: git bisect start Alternatív parancs: git bisect run Ne felejtsd el befejezni a bisect-et a git bisect reset paranccsal.
git reflog Visszakeresheted a korábbi git műveleteket. Reflog lekérése: git reflog Ha el akarod dobni a hibás állapotot, használhatod a git reset --hard-ot. Használd körültekintően, hogy ne veszíts el adatokat.
git archive Archívumot készíthetsz a repository-ból. Archívum létrehozása: git archive --format=zip HEAD > project.zip Használhatod a git archive --output kapcsolót is. Ne használj archívumot fájlok törléséhez, az csak a fájlok mentésére szolgál.
git submodule Almodulok kezelése egy repository-ban. Almodul hozzáadása: git submodule add <repository-url> Frissítés: git submodule update Ne felejtsd el a submodule-kat a git submodule init parancsokkal inicializálni.
git cherry-pick Lehetővé teszi egy commit alkalmazását egy másik ágra. Commit másolása: git cherry-pick <commit-hash> Használhatod a git cherry-pick --abort parancsot a folyamat megszakítására. Ha a commitokat nem jól választod ki, akkor konfliktusokhoz vezethetnek.

Docker Parancsok – Lépésről lépésre

Docker Parancs Mit csinál Példa Alternatíva / Megjegyzés Gyakori Hiba / Figyelmeztetés
docker --version Megjeleníti a Docker verzióját. docker --version Használható: docker version is Régi verzió esetén inkompatibilitási hibák lehetnek.
docker pull Letölti az adott image-et a Docker Hub-ról. docker pull nginx Privát repository esetén login szükséges. Letöltés sikertelen lehet, ha nincs internet vagy jogosultság.
docker images Kilistázza a letöltött image-eket. docker images Használható: docker image ls Túl sok image gyorsan elfogyaszthatja a tárhelyet.
docker ps Kilistázza a futó konténereket. docker ps Használható: docker container ls Csak a futó konténereket mutatja, a leállítottakat nem.
docker ps -a Kilistázza az összes konténert (futó és leállított is). docker ps -a Használható: docker container ls -a Sok leállított konténer rendetlenséget okozhat.
docker run Elindít egy új konténert az adott image-ből. docker run -d -p 80:80 nginx Használható --rm opcióval automatikus törléshez. Port ütközés, ha az adott port már foglalt.
docker exec Parancsot futtat egy futó konténeren belül. docker exec -it my_container bash Használható: docker attach A konténernek futnia kell a használathoz.
docker stop Leállít egy futó konténert. docker stop my_container Használható: docker kill Nem áll le azonnal, ha a konténer épp folyamatot végez.
docker start Elindít egy leállított konténert. docker start my_container Újraindításhoz: docker restart A konténernek léteznie kell, különben hibát dob.
docker rm Töröl egy konténert. docker rm my_container Több törléshez: docker rm $(docker ps -a -q) Futó konténer nem törölhető, előbb meg kell állítani.
docker rmi Töröl egy image-et. docker rmi nginx Használható: docker image rm Ha konténer használja az image-et, nem törölhető.
docker build Image-et készít egy Dockerfile alapján. docker build -t my_image . Dockerfile-nak a mappában kell lennie. Hiányzó vagy hibás Dockerfile hibát okoz.
docker logs Megmutatja egy konténer naplóit. docker logs my_container Használható: -f opció a folyamatos naplózáshoz. Napló túl hosszú lehet, nehéz olvasni.
docker-compose up Elindítja a docker-compose.yml-ben definiált konténereket. docker-compose up -d Használható interaktív módon is. Hibás YAML szintaxis megakadályozza az indulást.
docker-compose down Leállítja és eltávolítja a docker-compose konténereket. docker-compose down Volume törléshez: --volumes Volume-ok és hálózatok nem törlődnek automatikusan.
